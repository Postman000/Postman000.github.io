## 💼 实习经历

### 上海奥戴斯创车辆科技 · 底层系统软件实习生
*2026.01 - 2026.04*

#### 项目一：Bao Hypervisor 车规级多核移植
**技术栈**：C / RH850 汇编 / IAR / Type-1 Hypervisor / MPU 隔离

独立完成开源 Type-1 Hypervisor **"Bao"** 从 GCC 到 IAR 工具链的全栈移植，目标平台为 **Renesas RH850 U2A16** 四核车规 MCU，成功实现 Guest VM 的 MPU 隔离运行。

- **异常处理框架重写**：定位 IAR 不支持 GCC 宏内联导致的链接寄存器（r31）覆盖问题，通过逐指令反汇编对比确认根因，手动内联重写 `exceptions.S`，实现多核首次稳定运行
- **编译器优化冲突修复**：追踪 IAR 优化器处理复合字面量时的寄存器生命周期冲突，导致 MPU 区域大小被清零。通过拆解赋值语句引入序列点消除冲突，MPU 保护正确生效
- **多核启动竞态消除**：替换 IAR 初始化函数后从核在主核完成 `.bss` 清零前即进入 C 环境，多核同步死锁。在 `boot.S` 中引入汇编级硬屏障（魔数轮询），从根本上保证多核进入 C 环境的时序安全
- **ABI 环境补全**：IAR 默认启用的 SDA/TDA 基址寻址依赖 GP/TP 寄存器，但自定义引导代码未初始化，导致全局变量访问指向 `0x0` 触发总线异常。在 `boot.S` 中补全链接器符号加载，打通 IAR 完整 ABI 契约

#### 项目二：IAR Hypervisor 运行时监控插件开发
**技术栈**：C / IAR C-SPY SDK / 调试器插件开发

深入研究 IAR C-SPY Debugger 架构，基于官方 C-SPY SDK 独立开发针对 Bao Hypervisor 的专属调试辅助工具：

- 通过调用 SDK 底层内存与寄存器访问 API，实现 vCPU 核心系统寄存器（EIPC/FEPC/PSW 等）的**实时监控与界面可视化**
- 支持 **MPU 内存保护区域映射的图形化展示**，大幅提升 Hypervisor 调试效率

## 🚀 项目经历

### 基于 nRF54L15 的低功耗心电信号采集系统
**技术栈**：Zephyr RTOS / BLE / Qt 6 QML / 四层 PCB / 信号调理

设计一套可穿戴级心电信号采集系统，核心挑战是在低功耗 BLE SoC 上实现连续、无丢失的微伏级信号采集与实时传输。

- **硬件设计**：独立完成四层 PCB 设计，构建仪表放大器 + 带通滤波的模拟前端，实现微伏级心电信号采集
- **固件开发**：基于 Zephyr RTOS，采用 SAADC 14-bit / 500 sps + TIMER + PPI 硬件联动触发实现**零 CPU 开销**的连续采样，配合双缓冲机制消除数据丢失
- **无线传输**：设计自定义 BLE GATT 服务实时流式传输采样数据，单包携带 80 个 int16 采样点，支持丢包检测与序列号校验
- **驱动开发**：编写 **MCP4728 DAC Zephyr 驱动**（含 devicetree binding），用于偏置电压精密控制
- **上位机开发**：开发 **Qt 6 QML Android 上位机**，实现 BLE 实时波形显示、自动缩放坐标轴及过零检测频率分析
